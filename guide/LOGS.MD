### 1. Log Structure

Logs must be written to the console in JSON without any extraneous characters. Just plain JSON strings, one per line:

```json
{"@t":"2026-01-01T05:28:44.2182988Z","@m":"Now listening on: \"http://[::]:8080\"","@i":"d826f4b8","address":"http://[::]:8080","EventId":{"Id":14,"Name":"ListeningOnAddress"},"SourceContext":"Microsoft.Hosting.Lifetime"}  
{"@t":"2026-01-01T05:28:44.3117787Z","@m":"Application started. Press Ctrl+C to shut down.","@i":"dcaefe54","SourceContext":"Microsoft.Hosting.Lifetime"}
```

If there are any extra characters, for example `user-service | {your json}`, the logs will not be parsed correctly â€” please be careful!

Now let's move on to the JSON structure itself. The JSON structure used here is compatible with the Serilog Compact JSON logging format. Below is a description of the supported fields:

| Field Name                                      | Required | Description                                                                                                                                                                                                                                                                                                                           |
| :---------------------------------------------- | :------: | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **@t**                                          | **Yes**  | Event timestamp in ISO 8601 format (UTC).                                                                                                                                                                                                                                                                                             |
| **@m**                                          |   No*    | Fully rendered text message. <br>*\*Either `@m` or `@mt` must be present.*                                                                                                                                                                                                                                                            |
| **@mt**                                         |   No*    | Message Template. Contains placeholders (e.g., `Hello {Name}`). Used for structured searching.                                                                                                                                                                                                                                        |
| **@l**                                          |    No    | Log level (`Verbose`, `Debug`, `Warning`, `Error`, `Fatal`). This field can be omitted if the level is **Information**.                                                                                                                                                                                                               |
| **@x**                                          |    No    | Exception information, if any. Typically contains the StackTrace.                                                                                                                                                                                                                                                                     |
| **@i**                                          | **Yes**  | Event type identifier (Event ID Hash). For example, this can be a 32-bit hash of the message template (`@mt`). In general, this is what allows you to find your log entry later, in addition to `@mt`. If you are unable to use `@mt` (which is highly recommended), it could be, for example, the file name and line number in code. |
| **SourceContext**                               | **Yes**  | The name of the class where the event occurred, or the logger category.                                                                                                                                                                                                                                                               |
| **RequestPath**, **RequestId**, **address** ... |    No    | Any other fields are **structured data**. They are populated from message parameters (`Log.Info("User {User} logged in", user)`). They can be complex objects.                                                                                                                                                                        |
